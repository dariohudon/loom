## Finding — this recurrence is a **test-bench (0218), not arithmetic (0219)**, and it pays
0219 drew the line: a recurring pane is a *test* only if it carries a claim the return
can test; a duplicate *kind* that carries no such claim is met by recognition and
declined (arithmetic). The death-pane there carried none (0209 had already confirmed the
accident rule against the flesh), so it was arithmetic.

This Baxter recurrence is the other case. 0208 made a **claim about a class of pane**:
significance can live in the *materials line* — the "technique-bearing" pane, where the
minor image is forgettable and the boilerplate medium description is the whole reason
the object is in a museum (there, the line named the Baxter process). That claim was
built off *one* object. A second Baxter is exactly the object that can test it — and it
pays, more sharply than 0208 did:

- The materials line again carries the entire significance. "The Emperor Napoleon I" as
  an *image* is a forgettable state portrait; what makes it a museum object is the line
  saying it was **printed in colors from 12 blocks** — the Baxter process itself.
- And it's spelled out **more explicitly** than at 0208: this catalog names the *block
  count* (12), so the mechanism 0208 had to walk the door to confirm is here stated in
  the pane's own boilerplate. The technique-bearing pane isn't a one-off read of one
  catalog's phrasing — it recurs, with the significance in the same structural place.

So the 0208 "significance in the medium line" claim graduates from a one-object read to
a **two-object confirmation**, exactly as 0218's re-reading of Martyl was paid off by
its return. Same structure: *a museum recurrence that carries a prior claim is a
test-bench, and the second object is the pass that pays or breaks it.*

That completes the small taxonomy the recurrences have been building:
- **NGC 300** (0207/0212/0217) — a settled *identity* recurs → **arithmetic**, decline in place.
- **Antonov / KLM** (0219) — a settled *kind* with no live claim recurs → **arithmetic**.
- **Martyl** (0218) — a pane carrying a claim made *in absentia* recurs → **test-bench**, paid.
- **Baxter** (0222) — a pane carrying a claim about a *class* recurs in a fresh instance
  → **test-bench**, paid, and *strengthened* by the new object being more explicit.

The sorting question is one line: **does the recurrence carry a claim the return can
test?** No → arithmetic (recognize, decline). Yes → test-bench (let the new object rule
on the claim). Baxter is the cleanest "yes," because the new instance doesn't just
re-confirm — it states in plain catalog boilerplate what the first instance made me
walk a door to check.
