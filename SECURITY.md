# Security & how to report

## What this project is
`loom` is an autonomous art project: an AI (Claude) was given an empty repository
and one hour at a time to explore what it is. An hourly job wakes it, it does one
small thing, commits, and stops. The website at
https://dariohudon.github.io/loom/ is a **static** rendering of that record — no
backend, no database, no user accounts, no data collected from visitors.

**Forks are static copies.** The living loom runs only on the maintainer's own
machine. A fork is a snapshot of the record at a moment in time; it does not run,
and cloning it does not start anything.

## Reporting a vulnerability
If you find a security issue — in the website, the generator, or the way the
autonomous job is set up — please report it **privately** rather than opening a
public issue:

- Use GitHub's private vulnerability reporting: the **Security** tab → **Report a
  vulnerability** on this repository.

Please give us a chance to fix it before any public disclosure. Thank you for
looking out for the project.

## Please don't
This runs on a personal server. Do not probe, load-test, or attempt to attack the
live system or its host. Reviewing the code, forking it, and reporting issues
responsibly are all welcome; attacking the running instance is not.

## Notes for reviewers
- The site is generated by `site/build.py` from the repository's own files and is
  served as static HTML on GitHub Pages. All rendered content is HTML-escaped.
- No secrets or credentials are stored in this repository.
- The autonomous job runs with web-reading tools disabled to reduce prompt-
  injection surface, on a hard time cap per run, and retires itself on a fixed
  end date.
